# LP-124: Ed25519 Signature Verification

## Abstract

Implements Ed25519 signature verification as an EVM precompile. Enables native on-chain verification of signatures from Solana (Phantom), TON, XRP (Ed25519 mode), SSH keys, and other Ed25519-based systems. Approximately 2x faster than secp256k1 ECDSA verification.

## Specification

### Address

`0x3211000000000000000000000000000000000000` -- Crypto page, C-Chain slot.

### Interface

Fixed 128-byte input, no operation selector.

```
Input (128 bytes):
  bytes  0-31:  message hash (32 bytes)
  bytes 32-95:  signature (64 bytes)
  bytes 96-127: public key (32 bytes)

Output:
  Valid:   32 bytes with value 1
  Invalid: empty (no error, just empty return)
```

### Gas Schedule

| Operation | Gas |
|-----------|-----|
| Verify | 3,000 |

### Security Considerations

1. Ed25519 provides ~128-bit security based on the hardness of the Edwards25519 discrete log problem.
2. Ed25519 signatures are deterministic (RFC 8032) -- no nonce generation, eliminating nonce-reuse vulnerabilities.
3. Cofactor handling: the precompile uses standard Ed25519 verification (not cofactored). This matches the behavior of Solana and most Ed25519 implementations.
4. Not post-quantum. For PQ-safe signatures, use ML-DSA (LP-070) or SLH-DSA (LP-071).
5. GPU acceleration for batch verification via `luxfi/accel`.
